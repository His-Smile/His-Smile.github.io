空指针报错

“1”.equals（xxx）和（xxx）.enquals”1”

这两种写法在逻辑上是等价的，都会根据 jcShopEnquiry.getDataSource() 的值是否等于 “1” 来设置 request 的 enquiryType 属性为 “1” 或 “2”。但它们的区别在于代码的可读性和潜在的空指针风险：

  1. “1”.equals(jcShopEnquiry.getDataSource())
    优点：这种写法可以避免 jcShopEnquiry.getDataSource() 为 null 时抛出 NullPointerException，因为 “1” 是一个常量，不会为 null。
    推荐场景：当 jcShopEnquiry.getDataSource() 可能为 null 时，推荐使用这种写法。
  2. jcShopEnquiry.getDataSource().equals(“1”)
    缺点：如果 jcShopEnquiry.getDataSource() 为 null，会抛出 NullPointerException。
    推荐场景：当可以确保 jcShopEnquiry.getDataSource() 不会为 null 时，可以使用这种写法。
    总结
    如果 jcShopEnquiry.getDataSource() 可能为 null，使用第一种写法更安全。
    如果可以确保 jcShopEnquiry.getDataSource() 不为 null，两种写法都可以，但第一种写法更通用。
