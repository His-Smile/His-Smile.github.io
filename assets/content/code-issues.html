2.String.join

String result = String.join(“, “, “apple”, “banana”, “cherry”);
System.out.println(result); // 输出: apple, banana, cherry

如果 elements 中包含 null 值，String.join 会忽略这些 null 值，不会抛出异常。

String result = String.join(“, “, “apple”, null, “cherry”);
System.out.println(result); // 输出: apple, cherry

分隔符可以为空字符串 (“”)，但不能为 null，否则会抛出 NullPointerException。
如果 elements 为 null 或者是空数组/列表，则返回空字符串 (“”)。

VARCHAR

存储容量:
VAR-CHAR(n): 其中 n 是字符的最大长度，范围通常是 1 到 65,535（具体范围取决于数据库系统）。
例如，VARCHAR(255) 可以存储最多 255 个字符。
存储方式:
VARCHAR 只存储实际输入的字符数加上一个额外的字节（或两个字节，具体取决于数据库系统）来记录字符串的长度。
这意味着 VARCHAR 是可变长度的，存储效率较高，因为不会浪费空间。
使用场景:
适用于存储长度可变但相对较小的字符串，例如用户名、地址、描述等。

TEXT

存储容量:
TEXT: 存储容量通常非常大，具体取决于数据库系统。
例如，在 MySQL 中，

TEXT 类型的最大长度为 65,535 字节（64 KB）

LONGTEXT 可以存储最多 4,294,967,295 字节（4 GB）。
存储方式:
TEXT 类型通常存储在表之外的单独存储区域，表中只存储指向这些数据的指针。
这意味着 TEXT 是不可变长度的，存储效率较低，因为需要额外的存储空间来管理指针。
使用场景:
适用于存储非常长的文本数据，例如文章、日志、文档等。
