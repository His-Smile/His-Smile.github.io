接口

接口的定义

当一个类中的所有的方法都是抽象方法的时候，就可以将其定义为接口，接口也是一种引用数据类型，比抽象类更加的抽象

接口存在的两个重要的意义

​ 1.规则的定义

​ 2.程序的扩展性

接口用关键字interface来定义

接口不能实例化,接口可以同时实现两个，这两个接口里面可以用相同的抽象方法，因为实现只有在实现类中

接口中的成员的特点 default 和static

1.接口中的成员变量，是默认用finall进行修饰的，要进行大写

2.接口名.NUM也可成功调用 接口的成员变量默认被public static finall 修饰

jdk8之后，允许接口定义非抽象方法，但是需要用关键字default进行修饰，这个方法就是默认方法

public default void show(){} 这个时候，实现这个接口的不用强制实现这个方法但是，此接口实现类可以直接实例化之后调用这个方法。同时也可以重写，重写的实现类，重写默认方法，但是不能够加上default，可是如果遇到实现多个类，同一个方法，逻辑冲突，就必须进行重写

接口的默认方法所存在的作用就是解决接口升级，如果有需要在接口里面单独写方法的时候，保证丰富接口功能的同时不需要修改子类方法

**从jdk9开始，接口类可以使用private私有成员方法 ** private void show() private static void show()

静态方法只能够调用静态的

如果继承的和实现的，用相同的方法，优先使用继承的

接口和接口之间是继承关系，只有类可以实现，接口可以多继承

多态

同一个对象，在不同时刻表现出来的不同的形态，

多态的前提和体现：

​ 有继承或者实现的关系

​ 有方法重写

​ 有父类引用指向子类对象

多态中，成员访问的特点

1.成员变量：编译和运行用的父类的，但是如果父类当中没有写这个值的话，代码在编译阶段就直接报错了

​ 编译期间看左边，运行看左边，就是父类

2.成员方法： 编译期间看左边（父类），运行期间看右边（子类）

多态的转型：

1.向上转型 Fu = new ZI

2.向下转型 ZI z = (ZI) Fu

**ClassCastException ** （类型转换异常） 父类引用指向的子类对象。在父类强转之后，再引用另外的子类的方法就会报错

避免出现强转的风险就用这个

关键字 instanceof 类型

通俗的理解：判断关键字左边的变量，是否是右边变量的类型，返回boolean类型的结果
