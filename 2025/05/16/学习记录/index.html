关于IO流

1.输入输出的时候有字符编码，根本就是 阿斯克码和gbk，gbk一个中文两个字节，utf-8一个中文三个字节，还有是负数的时候就代表是中文了

2.关于序列化的时候，output，的时候是序列化，input的时候是反序列化

3.还有序列化的时候有个序列化码，arrayList里面就有，这个要一直，否则变动了接受类的时候就反序列化不上了

4.还有在使用流的时候要注意flush 刷可以用这个往后写，不清空，close是直接关闭不续写。需要弄到try catch 里面，在final里面也需要 try { final{try { ios.close}}}这样子

关于线程

1.继承Mythred (extend) run 和 star的区别，前者表示创建对象，用对象去调用方法，并没有去创建线程， 后者 star会创建线程 这里面源码有native 表示用的本地方法，开启线程，再由jvm虚拟机去调用run方法。

2.多线程的实现方式，实现Runnable ，创建muthred的对象，重写run（）方法， 然后创建thread 然后，将创建的对象，仿作对象放进去，调用star方法

  1. 实现Callable 和 future 重写call（）方法 然后创建future类的对象，把muable对象作为构造参数的方法放进去，

总结 ，三种方式 1.实现runbale 或者 CALLABLE

​ 2.继承thread

线程是由名字的，默认是 Thread-编号，getpro 线程优先级，默认是5，范围是1-10，但是优先级不一定是都是他先执行，而是概率比较高

CurrentHashMap1.7

16个数组，只有0索引创建一个 长度为2的小数组，把地址值赋值给0索引，其他的索引都是null

1.根据建的哈希值计算出segment（大数组）数组的索引

2.如果为空，就创建一个默认为2的数组

3.根据建的哈希值计算出在小数组应该存入的索引（二次哈希）

4.如果为空，就存入，如果不是空，就挂在下面，桶元素，如果1索引也是null就会进行扩容然后再存入元素，

大数组无法扩容，只有小数组（hashEntey[]）会扩容,一个线程进来，只能锁小表，不锁大表
